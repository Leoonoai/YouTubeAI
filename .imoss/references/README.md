# .imoss/references — 外部参考原文集中区

> 归属于 [任务 002 项目资料中心](../tasks/active/002-project-docs/plan.md)（living task，永不归档）

## 定位

**这里存放所有用户提供的外部参考原文** —— 视频文案转写 / 文章 / 书籍摘录 / 对谈实录 / 调研输入 / 外部数据集说明等，格式不限。任何 **YouTubeAI** 相关的"别人写的东西"或"用户手上的原始材料"都沉淀到这里，供：

- 未来任务 grep / 直接 Read
- 新 agent 快速了解 YouTubeAI 吸收过哪些素材
- 跨任务交叉引用（同一作者多篇、同一主题多视角）
- 避免每次新任务都在自己的 task dir 里留一份副本

## 规则

1. **文件命名**：`YYYY-MM-DD-<作者或主题>-<具体主题>.md`。示例：
   - `2026-04-11-张三-产品定位心法.md`
   - `2026-04-12-无作者-后端监控实践.md`
2. **格式不限**：视频文案转写、公众号文章、B 站字幕、对谈记录、调研 transcript、外部 API 文档片段——统一用 Markdown 存
3. **原文保留**：不要修改或美化原文（包括语音识别错漏），保持原样；结构化吸收另外放 `.imoss/knowledge/lessons/`
4. **任务里不重复存**：新任务要分析某份原文，**task dir 里不放 source-article.md**，改为在 plan.md 里引用 `.imoss/references/YYYY-MM-DD-...md` 路径
5. **历史任务不追溯**：已归档任务里的 source-article.md 保持原位（committed history 不动），在下方「历史引用」表里链接过去即可
6. **索引及时更新**：每次新增原文，在下方「索引」表里加一行
7. **只装原文不装总结**：总结和 distilled lesson 走 `.imoss/knowledge/lessons/`，不混淆
